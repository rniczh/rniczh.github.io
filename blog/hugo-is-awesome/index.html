Hugo + Theme

Install Hugo

這部份可以看Hugo-Basic-Usage了解，因為我是用Arch Linux，所以就直接用Pacman安裝就好，安裝好後找一個地方放Blog

$ sudo pacman -S hugo
$ hugo new site /path/to/your/blog

然後就可以在那目錄下找到以下內容：

▹   default.md : Post的基本樣板
▹   config.toml : 網站的配置文件
▸   content/ : 文章放置處
▸   data/ : data-templates
▸   layouts/ : 網站的layout
▸   static/ : 靜態資源(e.g. image file)
▸   themes/ : Themes

Setup Theme

這裡以Hugo-Primer這個Theme為例，只有兩個步驟：

  1. 將對方的Theme Clone下來放進你的themes目錄下

    $ git clone https://github.com/qqhann/hugo-primer.git themes/hugo-primer

  2. 將此行設定至 Config.toml

    theme = "hugo-primer"

Preview Blog

此時就可以嘗試去預覽Blog的情況：

$ hugo server -D -F

-D 代表Draft，在文章中的Params那設定 Draft: true 的文章必須用 -D 來顯示， 這些文章在最後並不會Published上去，除非把它設定為 false。

-F 代表 Future，因為PublishedDate通常會是預定要發布的時間，所以都是”未來”，用這個方式才能在預覽時顯示未來的文章。

然後在 localhost:1313/ 就看到Blog目前的樣子了。

Hugo Config

設定方面指的是Config.toml的設定，由於在Hugo-Primer那Readme已經有寫了Config的部份， 這裡就只會針對他沒寫到的部份增加而已，而我自己有Fork了他的一份做修改，可以在Hugo-Primer-edited找到。

  1. Paginate 這個變數是決定一頁要有多少文章，預設是10

    # Default = 10. This setting can be overridden within the template.
# It's indicate how many posts can be placed in one page
# detail: https://gohugo.io/templates/pagination/
Paginate = 10

  2. Menu設定

    那個Theme提供的範例圖片右上角會有一個Menu就是在這裡設定，透過url來指定他在網址的位置，例如我把Info的部份設定到 / 主目錄下， 所以連到我的Blog https://rniczh.github.io/ 會看到的是我的個人介紹，而 /blog/ 則是Blog的文章列表， 那接下來的問題就會是，怎麼顯示那個Page，其中 content/ 指向的url是 / ，所以再根據Hugo的規則

    • 新增 content/_index.md，則就會建立一個 url: / 下的頁面了

    • 相同地，新增 content/blog/_index.md 建立的就是 url: /blog/ 下的頁面

    [[menu.main]]
  identifier = "blog"
  name = "Blog"
  title = "blog section"
  url = "/blog/"
  # weight is indicate the display order in menu
  weight = -100

[[menu.main]]
  identifier = "info"
  name = "Info"
  title = "info section"
  url = "/"
  weight = 50

    _index.md will generate index.html

之後寫文章就寫在content/blog/下，或者你不喜歡blog這樣的名稱，也可以改成posts之類的，只要記得相對應的設定記得改就好。

Emacs Config

來到這邊的你其實可以回頭了，除非也是要來設定Emacs的xD

這邊有兩個工具（easy-hugo + ox-hugo）要使用，大致上處理流程如下圖

Easy-Hugo

easy-hugo我只是拿來管理文章而已，例如確認哪些目前是Draft的情形等等之類的，實際上發布或者預覽都是用shell上的hugo指令完成。

而easy-hugo設定舉 github.io 為例：

(use-package easy-hugo
  :config
  (setq easy-hugo-basedir "path/to/your/blog")
  (setq easy-hugo-url "https://yourname.github.io")
  (setq easy-hugo-postdir "content/blog/")
  (setq easy-hugo-sshdomain "yourname.github.io")
  (setq easy-hugo-root "/home/rnic/Project/githubio/hugoroot/")
  (setq easy-hugo-previewtime "300")
)

I’m use `use-package` to manage package in Emacs

Ox-Hugo

ox-hugo，這是我覺得可能會比較複雜一點的，不過他的設定很簡單：

(use-package ox-hugo
  :after ox)

比較複雜的是他的文章表示（下面的Org File），因此我也把它加進了Emacs的yasnippet裡，省的我複製樣板的麻煩， ox-hugo的作用是把 📖 Org file轉換成 🗒 Markdown file，因為Hugo在Org上有些問題，所以變成這樣使用可以不必踩那些坑， 不過你就得先踩Emacs這個大坑 (

#+HUGO_BASE_DIR: ../
#+HUGO_SECTION: ../content/blog/
#+HUGO_WEIGHT: auto
#+HUGO_AUTO_SET_LASTMOD: t
* AnyName                                                             :@category:
  The AnyName does not affect the website, so you can set it to any name you like.
  All posts in here will have the category set to /category/
  and you need type C-c C-s on this title block to generate publishedDate
** TODO Title                                                         :tag1:tag2:
   :PROPERTIES:
   :EXPORT_FILE_NAME: your_filename
   :EXPORT_HUGO_CUSTOM_FRONT_MATTER: :comments true
   :END:
   Abstract
#+hugo: more
*** First headline here
	Content

* Footnotes
* COMMENT Local Variables                          :ARCHIVE:
  # Local Variables:
  # eavl: (org-hugo-auto-export-mode)
# End:

作法是在主目錄下除了content以外，再多新增一個目錄叫做content-org(其他名子也可以) 然後再新增一個Org file template.org ，並複製上面那個內容進去。

$ mkdir content-org
$ touch content-org/template.org

之後就依照上面那個範本進行寫作，像是你轉換過後的Markdown File要去哪裡、他的檔名又是什麼之類的，其中有幾點要注意的是

  1. 在 TODO Title 那個Block內敲下 C-c C-s ，可以標注PublishedDate，這是一定要標的！

  2. 在 TODO Title 那個Block內敲下 C-c C-e H H 會把這份文件轉換成Markdown，並以 your_filename.md 寫進 content/blog/ 目錄內

  3. 或者是利用上面內容中最下面的 eval: (org-hugo-auto-export-mode) 讓存檔時就自動轉換過去

  4. 把 TODO 刪掉後則對應在Markdown就是 Draft: false ，當然，改成 DONE 也有同樣效果

這樣用ox-hugo寫文章有什麼優點？

  1. 可以用Org 😁

  2. 在同一個文件裡，其實可以寫成一系列的文章，而不用再建立的文件，前後對照速度也就會快很多
